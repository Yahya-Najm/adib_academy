PHASE 1 — SCHEMA CHANGES                                              [DONE ✅]
════════════════════════════════════════════════════════════════════════════════

Files changed:
  - prisma/schema.prisma

What was done:
  - Added ReportKind enum: SIMPLE | ACTIONABLE
  - Added ExamType enum: REGULAR | FINAL
  - Added `userId` field (String? @unique) to User model
      → slug-style auto-generated ID e.g. "john-doe-a4f2"
      → applies to all roles: Manager, Teacher, Staff
  - Updated AttendanceReport model:
      → added reportKind (ReportKind, default SIMPLE)
      → added isDone (Boolean?) — null for SIMPLE, false/true for ACTIONABLE
      → added doneAt (DateTime?)
      → added doneById (String?) + doneBy relation
      → added actionDescription (String?) — what the manager needs to do
  - Updated Exam model:
      → added examType (ExamType, default REGULAR)
      → added classMonth (Int?) — auto-detected month of the class the exam falls in
      → added proctorId (String?) + proctor relation — teacher/manager who supervised
      → added scoringFinalized (Boolean, default false)
      → added scoringFinalizedAt (DateTime?)
      → added scoringFinalizedById (String?) + scoringFinalizedBy relation
  - Changed AttendanceFinalization unique key:
      → OLD: @@unique([date, branchId])
      → NEW: @@unique([date, branchId, finalizationType, scopeId])
      → finalizationType: "STAFF" | "TEACHER" | "CLASS"
      → scopeId: the userId/teacherId/courseClassId being finalized
      → allows per-person and per-class finalization instead of one branch-wide lock
  - Added ClassMonthFinalization model:
      → courseClassId, monthNumber, branchId, finalizedById
      → sectionsSnapshot (Json) — captures teacher-per-section at finalization time
      → studentsSnapshot (Json) — captures enrolled students at finalization time
      → @@unique([courseClassId, monthNumber])

DESIGN CONVENTIONS (for reference)
================================================================================

  Colors:
    - GM accent: orange
    - Manager accent: teal
    - Teacher accent: dark/gray
    - PAID: green | PENDING: yellow | OVERDUE: red | PARTIAL: orange
    - Final exam: purple | Regular exam: blue
    - Actionable report pending: orange | done: gray

  Page structure:
    - bg-white pages, bg-gray-50 content area
    - Cards: bg-white border border-gray-200 rounded-xl shadow-sm
    - All mutations via Server Actions ("use server")
    - Guards: requireGM() / requireManager() / requireTeacher() at top of every action
    - Client pages use useTransition for server action calls

  ID format:
    - Students: firstname-lastname-xxxx (studentId on Student model)
    - Users (teachers/staff/managers): firstname-lastname-xxxx (userId on User model)
